# 需求

## 登陆/注册

- [ ] 注册：输入用户名和密码进行注册，注册成功后返回用户 id，失败时提示创建失败。
- [ ] 登录：输入用户名和密码登录，成功后返回 token 供后续使用，失败时提示用户不存在或密码错误。
- [ ] 更新用户信息：登录后携带 token 提交新的用户名和密码进行更新，成功时提示更新完成；若 token 无效、过期或或用户名不匹配则提示重新登录或校验失败。

## Agent 模块

### 1. 行程管理

- [ ] 创建行程：登录后可以新建一个行程，创建时可选填行程名称，不填则由后端自动生成；创建成功后返回行程 ID，后续所有消息和操作都基于该行程。一个行程包含一个旅行计划（初始为空）以及与 Agent 的对话历史。
- [ ] 获取行程列表：登录后可以查看当前用户创建的所有行程，按更新时间倒序排列。
- [ ] 获取行程详情：登录后可以查看指定行程的完整信息，包括旅行计划和对话历史。
- [ ] 删除行程：可以删除不需要的行程，一并清除该行程的所有消息和关联的旅行计划。

### 2. 旅行计划

每个行程对应一个旅行计划，记录行程的详细安排，用户和 Agent 都可以对旅行计划进行编辑。

- [ ] 获取旅行计划：获取指定行程的当前旅行计划，包含景点、酒店、交通、餐厅等各项安排。
- [ ] 更新旅行计划：用户或 Agent 都可以更新旅行计划，支持添加、修改、删除计划项。更新时应携带操作类型（添加/修改/删除）和具体的计划内容。后端需记录操作来源（用户/Agent）以支持后续审计。

### 3. 消息交互

- [ ] 发送消息：向指定行程发送一条消息，支持多模态输入，包括文本、图片、语音、文档等格式。后端调用 Agent 处理并返回回复内容，若消息包含需要 Agent 调用工具的场景（如搜索景点、订票订酒店、添加旅行计划项等），后端应返回 Agent 调用了哪些工具及其结果；若使用流式响应，前端能够逐段展示 Agent 的思考过程和回复内容。
  多模态支持说明：
  输入：支持用户发送图片（景点照片、护照、票据等）、语音（语音输入指令）、文档（行程单 PDF 等）供 Agent 分析理解。
  输出（如果支持）：Agent 回复可包含图片（景点图片、地图截图等）、文档（行程单、票据等）等多模态内容。
- [ ] 获取历史消息：登录后可以查看指定行程的消息记录，按时间顺序展示用户消息和 Agent 回复，Agent 回复中应包含调用过的工具信息。每条消息都应有唯一 ID，方便前端界面设计"快速访问历史对话"的逻辑。
- [ ] 终止生成：当 Agent 正在生成回复时，用户可以主动终止生成，终止后返回已生成的部分内容。用户可以重新输入问题。

### 4. 错误处理与重试

- 错误类型：
  - `NETWORK_ERROR`：网络连接失败
  - `TIMEOUT`：请求超时（Agent 处理时间过长）
  - `AGENT_ERROR`：Agent 处理异常
  - `TOOL_CALL_FAILED`：工具调用失败（如外部 API 不可用）
  - `INVALID_REQUEST`：请求参数错误
  - `UNAUTHORIZED`：未授权（token 过期或无效）

- 重试策略：
  - 网络错误或超时时，前端应自动重试，建议重试间隔 1s、2s、4s，最多重试 3 次
  - 工具调用失败时，Agent 应尝试调用替代工具或返回部分结果，前端展示可用的结果
  - token 过期时，前端应提示用户重新登录
  - 对于流式响应中断的情况，前端应支持从断点恢复或重新发起请求
