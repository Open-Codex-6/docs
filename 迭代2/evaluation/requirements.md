# 需求

## 登陆/注册

- [ ] 注册：输入用户名和密码进行注册，注册成功后返回用户 id，失败时提示创建失败。
- [ ] 登录：输入用户名和密码登录，成功后返回 token 供后续使用，失败时提示用户不存在或密码错误。
- [ ] 更新用户信息：登录后携带 token 提交新的用户名和密码进行更新，成功时提示更新完成；若 token 无效、过期或与用户名不匹配则提示重新登录或校验失败。

## Agent评估

### 前端

- [ ] 导出报表/报告：输出整个任务的评估结果列表为文件形式

### 后端

#### 任务

- [ ] 创建任务，传入用户自定义的任务信息，结构大致包含以下信息：
  - 任务标题、任务简介、选中的指标id
- [ ] 按照创建时间获取所有的任务记录的简要信息，包括（但可能不限于）：
  - 任务 id、任务标题、任务简介、创建时间、评估次数、评估涉及指标
- [ ] 获取指定任务 id对应的所有评估记录简要信息，包括（但可能不限于）：
  - 评估记录编号（每个任务单独编号），评估创建时间、评估结果指标数据
- [ ] 修改任务信息
- [ ] 删除任务

#### 指标

- [ ] 创建指标，传入用户自定义的指标信息，结构大致包含以下信息：
  - 指标标题、指标简介、以及对应的 { 分数, 分数描述 }
- [ ] 获取系统设定的所有指标的具体信息，包括（但可能不限于）：
  - 指标 id、指标标题、指标简介
- [ ] 获取用户自定义的所有指标的具体信息，包括（但可能不限于）：
  - 指标 id、指标标题、指标简介
- [ ] 修改指标信息
- [ ] 删除指标

#### 评估

- [ ] 进行评估，传入任务 id 与 具体的评估数据集（是否保存便于后续再次查看待确定，暂定于不保存），返回该任务下该数据集的评估指标结果
